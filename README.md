
# eii-puzzles

A puzzle generator for edge matching puzzles (eg. Eternity II, Tetravex, GEMP-F). Hard generated instances of various sizes as similar as possible to the properties of the Eternity II puzzle. A C library for parsing, sorting and placing hints.

Edge matching puzzles consist of tiling all squares of an arbitrary sized chessboard with square tiles such that adjacent tiles have the same colored edges. The boarder of the chess board has a separate color (color *zero*) which indicates a subset of tiles which can touch the border.

## Listing
* **puzzles** generated puzzles.
* **real/eii.json** eternity ii puzzle instance using the original puzzle tile ordering to determine edge color values.
* **others** puzzles from outside sources.
* **scripts/gen-puzzles.sh** generate puzzles in *puzzles*.
*  **scripts/build.sh** build source code.
* **scripts/gen-real.sh** generate the eternity ii puzzle instance and check against others.

```
{

  "cols": 4,

    [0, 2, 1, 0],

  ],
  "placed": [

  ]

```

## Generating Algorithm
Puzzles are generated by selecting the number of edge colors across four categories of tile edges:
1. Corner tiles (corner-border edges -- CB).
2. Border tiles (border-border edges -- BB).
3. Border tiles (border-inside edges -- BI).
4. Inside tiles (inside-inside edges -- II).

The edges are assigned, then shuffled across all tiles.

Tile **edges are ordered**: north, east, south, west.

Five tiles are **placed as hints** in the middle and near the four corners of the board. Placed tiles are specified inline by their four edge colors to remain robust to tile re-reordering and clockwise vs counter-clockwise rotation.

The generated instance's tiles are in **solution order** row-by-row from top left. To prevent this from silently over-estimating your solver performance, use `eii-print --sort` to reorder tiles into lexicographically least  order.

See **src/tools/eii-gen.py**. 

## Puzzles

### Hard Instances
The hardest puzzles have the same number of colors as the size of the board. For example, an 8x8 board will have 8 inside-inside edge colors (ie. "0 offset"). The real eternity ii puzzle is of size 16x16 with 17 inside-inside colors (ie. "1 offset"). Both 0 and +1 offset puzzles are generated here to prevent an abrupt jump.

See **scripts/gen-puzzles.sh**.

## C Library

Generated puzzles can be parsed, printed and sorted using the header-only header library. Include both headers below in your project:
```
$ ls src/include/  
eii-puzzle.h jsmn.h
```

## Tools
### eii-print
Allows printing a puzzle instance. The instance can be processed into lexicographically least order (instead of solution order) by usign the `--sort` flag. 

Print a puzzle instance on the command line:
```
$ cat puzzles/4x4/4-2-4-4/eii-4x4-shuffle-4-2-4-4-17029.json | eii-print 
 0   0   0   0   
0:2 2:1 1:4 4:0  
 1   1   4   3   
 1   1   4   3   
0:2 2:4 4:3 3:0  
 2   3   1   2   
 2   3   1   2   
0:3 3:2 2:4 4:0  
 1   1   2   2   
 1   1   2   2   
0:3 3:1 1:4 4:0  
 0   0   0   0   
```

### eii-check
Calculate the number of edge conflicts if the tiles are placed row-by-row from the top-left.
```
$ cat puzzles/4x4/4-2-4-4/eii-4x4-shuffle-4-2-4-4-17029.json | eii-check
{"numEdgeConflicts": 0}
$ cat puzzles/4x4/4-2-4-4/eii-4x4-shuffle-4-2-4-4-17029.json | eii-print --json --sort | eii-check  
{"numEdgeConflicts": 53}
```

```

    [0, 0, 1, 1],

  ],  
  "placed": []  
}
```
